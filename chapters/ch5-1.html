معرفی Apache Kafka

📅 به‌روزرسانی: 1405/04/31
⏱️ زمان مطالعه: ۹ دقیقه
👤 تیم فنی

Apache Kafka یک پلتفرم Distributed Event Streaming است که برای انتقال، ذخیره و پردازش حجم بالایی از Eventها استفاده می‌شود.

سرویس‌های مختلف می‌توانند Eventهای خود را در Kafka منتشر کنند و سرویس‌های دیگر نیز آن‌ها را با سرعت و استقلال بیشتری دریافت و پردازش کنند.

🎯
Kafka فقط یک Message Queue ساده نیست؛ Eventها را برای مدت مشخص نگهداری می‌کند و Consumerها می‌توانند آن‌ها را دوباره بخوانند.

یک مثال ساده

فرض کنید پس از انجام موفق پرداخت، سرویس Payment رویداد PaymentCompleted را منتشر می‌کند.

Kafka Event Flow 

Payment Service

      │

PaymentCompleted

      ▼

Kafka Topic

      │

      ├──► Order Service
      │       Update Order Status
      │
      ├──► Notification Service
      │       Send Notification
      │
      └──► Analytics Service
              Record Transaction

هر Consumer می‌تواند Event را به‌صورت مستقل دریافت کند، بدون اینکه Payment Service از وجود آن‌ها اطلاع داشته باشد.

✅
اضافه شدن یک Consumer جدید معمولاً نیازی به تغییر Producer ندارد.

مفاهیم اصلی Kafka

مفهوم توضیح
Producer Event یا Message را در Kafka منتشر می‌کند.
Consumer Eventها را از Kafka دریافت و پردازش می‌کند.
Topic مسیر منطقی نگهداری Eventهای مرتبط است.
Partition Topic را به بخش‌های کوچک‌تر برای پردازش موازی تقسیم می‌کند.
Broker یکی از سرورهای Kafka است که داده‌ها را نگهداری می‌کند.
Offset شماره موقعیت هر Event داخل یک Partition است.
Kafka Structure 

Kafka Cluster

   ├── Broker 1
   ├── Broker 2
   └── Broker 3

Topic: payment-completed

   ├── Partition 0
   ├── Partition 1
   └── Partition 2

تفاوت Kafka با Message Queue سنتی

در بسیاری از Message Queueها، پیام پس از پردازش و تأیید Consumer حذف می‌شود.

اما Kafka Eventها را براساس مدت زمان یا حجم مشخص نگهداری می‌کند؛ حتی اگر قبلاً توسط Consumer خوانده شده باشند.

ویژگی Message Queue سنتی Kafka
نگهداری پیام معمولاً تا زمان پردازش براساس Retention Policy
خواندن مجدد معمولاً محدود امکان‌پذیر است
پردازش موازی با چند Consumer با Partition و Consumer Group
کاربرد اصلی پردازش Job و Command Event Streaming و پردازش داده
💡
Kafka برای Event Streaming، پردازش حجم بالا و نگهداری تاریخچه Eventها بسیار مناسب است.

نصب پکیج Kafka در .NET

یکی از کتابخانه‌های رایج برای کار با Kafka در .NET، پکیج Confluent.Kafka است.

.NET CLI 
dotnet add package Confluent.Kafka

ارسال Message با Producer در .NET

در مثال زیر، یک Event داخل Topic با نام payment-completed منتشر می‌شود.

C# Producer 
using Confluent.Kafka;
using System.Text.Json;

var config = new ProducerConfig
{
    BootstrapServers = "localhost:9092"
};

using var producer =
    new ProducerBuilder<string, string>(config).Build();

var paymentEvent = new
{
    PaymentId = 1205,
    OrderId = 1001,
    Amount = 850000,
    Status = "Completed",
    OccurredAt = DateTime.UtcNow
};

var message = new Message<string, string>
{
    Key = paymentEvent.OrderId.ToString(),
    Value = JsonSerializer.Serialize(paymentEvent)
};

var result = await producer.ProduceAsync(
    "payment-completed",
    message
);

Console.WriteLine(
    $"Message published to {result.TopicPartitionOffset}"
);

مقدار Key در انتخاب Partition نقش دارد. پیام‌هایی که Key یکسان دارند معمولاً در یک Partition قرار می‌گیرند.

📌
استفاده از OrderId به‌عنوان Key می‌تواند به حفظ ترتیب Eventهای مربوط به یک سفارش کمک کند.

دریافت Message با Consumer در .NET

C# Consumer 
using Confluent.Kafka;

var config = new ConsumerConfig
{
    BootstrapServers = "localhost:9092",
    GroupId = "order-service",
    AutoOffsetReset = AutoOffsetReset.Earliest
};

using var consumer =
    new ConsumerBuilder<string, string>(config).Build();

consumer.Subscribe("payment-completed");

try
{
    while (true)
    {
        var result = consumer.Consume();

        Console.WriteLine(
            $"Received: {result.Message.Value}"
        );

        // Process the event
    }
}
catch (OperationCanceledException)
{
    consumer.Close();
}

Consumer با استفاده از GroupId عضو یک Consumer Group می‌شود. Kafka پیام‌های Partitionها را میان اعضای همان Group تقسیم می‌کند.

⚠️
در پروژه واقعی باید خطا، CancellationToken، ثبت Offset و پردازش پیام تکراری به‌درستی مدیریت شوند.

Kafka چه کاربردهایی دارد؟

  • انتقال Event میان Microserviceها
  • ثبت و پردازش تراکنش‌های مالی
  • جمع‌آوری Log و داده‌های سیستمی
  • پردازش داده‌های Real-time
  • ساخت Data Pipeline
  • همگام‌سازی اطلاعات میان سیستم‌ها

مزایای Kafka

  • توان پردازش حجم بالای Event
  • مقیاس‌پذیری با استفاده از Partition
  • نگهداری Eventها برای خواندن مجدد
  • تحمل خطا با Replication
  • مناسب برای Event-Driven Architecture
  • امکان پردازش هم‌زمان توسط چند Consumer Group

چالش‌های Kafka

چالش توضیح
پیچیدگی عملیاتی راه‌اندازی و Monitoring آن نیازمند دانش مناسب است.
Ordering ترتیب فقط داخل یک Partition تضمین می‌شود.
Duplicate Message Consumer باید در برابر پردازش تکراری مقاوم باشد.
Partition Design انتخاب تعداد Partition و Message Key اهمیت زیادی دارد.
Schema Evolution تغییر ساختار Event باید با Consumerها سازگار باشد.

چه زمانی از Kafka استفاده کنیم؟

  • حجم Eventها بالا است.
  • نیاز به نگهداری و Replay کردن Eventها داریم.
  • چند سیستم باید یک Event را مستقل پردازش کنند.
  • پردازش Real-time یا Streaming موردنیاز است.
  • مقیاس‌پذیری و Throughput بالا اهمیت دارد.

چه زمانی Kafka انتخاب مناسبی نیست؟

  • سیستم کوچک و دارای تعداد کمی Message است.
  • فقط به یک Queue ساده برای اجرای Job نیاز داریم.
  • تیم توان نگهداری زیرساخت Kafka را ندارد.
  • پیچیدگی Kafka بیشتر از نیاز واقعی پروژه است.

اشتباه رایج

❌
انتخاب Kafka فقط به دلیل Performance بالا اشتباه است. قبل از انتخاب آن باید نیاز به Event Streaming، Retention، Replay و پردازش حجم بالا وجود داشته باشد.

نکته مصاحبه

🎓
سؤال: مهم‌ترین تفاوت Kafka با RabbitMQ چیست؟

پاسخ: RabbitMQ بیشتر برای Message Queue، Routing و پردازش Task مناسب است؛ اما Kafka برای Event Streaming، نگهداری تاریخچه Eventها، Replay و پردازش حجم بالای داده طراحی شده است.

قانون طلایی

🎯
Kafka را زمانی انتخاب کنید که Eventها بخشی از جریان داده سیستم باشند و نیاز به نگهداری، پردازش موازی یا خواندن مجدد آن‌ها وجود داشته باشد.

جمع‌بندی

Apache Kafka یک پلتفرم Distributed Event Streaming است که Producerها Eventها را داخل Topic منتشر می‌کنند و Consumerها آن‌ها را به‌صورت مستقل پردازش می‌کنند.

Kafka با استفاده از Topic، Partition، Offset و Consumer Group امکان پردازش حجم بالای Eventها، مقیاس‌پذیری و Replay داده‌ها را فراهم می‌کند.

➡️
در بخش بعد معماری Kafka را بررسی می‌کنیم و با Broker، Cluster، Topic، Partition و نحوه ذخیره Eventها آشنا می‌شویم.
