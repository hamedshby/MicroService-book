نصب RabbitMQ

📅 به‌روزرسانی: 1405/04/26
⏱️ زمان مطالعه: ۸ دقیقه
👤 تیم فنی

RabbitMQ را می‌توان روی ویندوز، لینوکس، Docker و Kubernetes اجرا کرد. امروزه در اکثر پروژه‌های Microservice، RabbitMQ داخل Docker اجرا می‌شود؛ زیرا راه‌اندازی، به‌روزرسانی و مدیریت آن بسیار ساده‌تر است.

🎯
اگر قصد یادگیری RabbitMQ را دارید، بهترین و سریع‌ترین روش استفاده از Docker است.

پیش‌نیازها

  • Docker
  • Docker Compose (اختیاری)
  • پورت‌های 5672 و 15672 آزاد باشند.

اجرای RabbitMQ با Docker

ساده‌ترین روش اجرای RabbitMQ استفاده از Image رسمی آن است.

Docker 
docker run -d \
--name rabbitmq \
-p 5672:5672 \
-p 15672:15672 \
rabbitmq:4-management

نسخه management علاوه بر RabbitMQ، پنل مدیریتی تحت وب را نیز در اختیار شما قرار می‌دهد.

✅
پس از اجرای Container، RabbitMQ آماده استفاده خواهد بود.

پورت‌های RabbitMQ

پورت کاربرد
5672 ارتباط برنامه‌ها با RabbitMQ از طریق AMQP
15672 پنل مدیریتی تحت وب

ورود به پنل مدیریت

پس از اجرای RabbitMQ، مرورگر را باز کرده و آدرس زیر را وارد کنید.

Management UI 
http://localhost:15672

اطلاعات پیش‌فرض ورود به صورت زیر است:

فیلد مقدار
Username guest
Password guest
⚠️
در محیط Production هرگز از نام کاربری و رمز عبور پیش‌فرض استفاده نکنید.

اشتباه رایج

❌
بسیاری تصور می‌کنند Producer مستقیماً پیام را داخل Queue قرار می‌دهد؛ در حالی که در RabbitMQ پیام ابتدا وارد Exchange شده و سپس به Queue مناسب هدایت می‌شود.

نکته مصاحبه

🎓
سؤال: چرا معمولاً از Image rabbitmq:management استفاده می‌شود؟

پاسخ: زیرا علاوه بر Broker، پنل مدیریتی تحت وب را نیز در اختیار قرار می‌دهد و مشاهده Queueها، Exchangeها، Connectionها و Consumerها را بسیار ساده می‌کند.

قانون طلایی

🎯
برای محیط توسعه همیشه RabbitMQ را با نسخه management اجرا کنید تا بتوانید جریان پیام‌ها را به‌صورت گرافیکی مشاهده و عیب‌یابی کنید.

جمع‌بندی

ساده‌ترین روش نصب RabbitMQ استفاده از Docker است. پس از اجرای Container، Broker روی پورت 5672 و پنل مدیریتی روی پورت 15672 در دسترس خواهد بود.

در ادامه این فصل با مفاهیم اصلی RabbitMQ مانند Exchange، Queue، Binding و Routing Key آشنا خواهیم شد.

➡️
در بخش بعد با مفاهیم اصلی RabbitMQ آشنا می‌شویم و بررسی می‌کنیم Exchange، Queue، Binding و Routing Key چه نقشی در انتقال پیام دارند.
